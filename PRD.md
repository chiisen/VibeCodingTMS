## 產品需求文件 (PRD)：TMS（運輸管理系統）Demo 網站

### 產品概述
本系統是一個運輸管理系統（TMS）的示範網站，主要用於規劃及管理貨物的運輸與配送路線。系統不使用 API 或資料庫，全部資料均以暫存或記憶體內的資料結構（如 Python dictionary、list）管理，方便展示系統關鍵功能及流程。

### 目標用戶
- 物流管理人員
- 供應鏈管理人員
- 運輸調度人員

### 核心功能
1. **貨物管理（暫存）**
   - 添加新貨物（貨物名稱、數量、重量等屬性）
   - 編輯貨物資訊
   - 刪除貨物
   - 顯示目前所有貨物列表

2. **運輸路線規劃**
   - 手動新增運輸路線（起點、終點、途經站點）
   - 編輯已有路線
   - 刪除路線
   - 路線列表顯示路線資訊及貨物配置

3. **貨物配送安排**
   - 指派貨物到特定運輸路線
   - 查看每條路線的分配貨物及總重

4. **管理介面**
   - 前端頁面操作界面（使用 Flask 路由與 Jinja2 模板）
   - 表單提交資料，後端使用 Python 變數儲存（不連接任何資料庫）

5. **系統狀態**
   - 顯示目前所有貨物、路線、配送狀況
   - 提供簡易搜尋（如根據貨物名稱或路線名稱過濾）

### 非功能需求
- 使用 Python Flask 開發，前端可用基礎 HTML/CSS/JS（可內嵌Bootstrap）
- 不使用資料庫及外部 API，所有資料存在伺服器記憶體
- 使用者操作體驗流暢，頁面切換簡潔明瞭
- 簡易響應式設計，適合桌面和手機預覽

### 預期系統架構
- Flask Web Server
- 靜態模板（Jinja2）呈現前端
- 資料以 Python 全域變數或 Session 管理
- 基本表單 CRUD（新增、修改、刪除）所有核心資料

### 發展步驟與範例頁面
1. 首頁：介紹系統、展示目前貨物與路線匯總
2. 貨物管理頁：貨物清單與新增/編輯/刪除功能
3. 路線管理頁：路線清單及操作功能
4. 配送管理頁：貨物指派與配送狀況
5. 簡單搜尋與過濾（前端或後端實作）
